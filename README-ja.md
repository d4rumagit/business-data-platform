# Small-Business-Operations-Platform-Project（小規模ビジネス運用プラットフォーム）
このプロジェクトの目標は**単なるアプリを構築すること**ではなく、以下のことです：
- **SQL、データモデリング、レポーティング**を学ぶ
- **Go を使用したバックエンド開発**を学ぶ
- **HTML と CSS を使用した基本的なフロントエンド開発**を実装する
- **企業の内部ツール**がどのように設計・使用されているかを理解する

このプロジェクトは以下を意識して設計されています：
- POSバックエンド
- 管理ダッシュボード
- 在庫管理システム
- 給与計算とスタッフ追跡ツール

---

## ☕︎ このプロジェクトが存在する理由
多くの企業には顧客向けアプリケーションがすでに存在します。
本当に必要で、維持が困難なのは**内部ツール**です。

このプロジェクトは以下の能力を示します：
- 構造化されたデータの設計
- ビジネスロジックの実装
- CRUD システムの構築
- SQL を使用したレポート生成
- フロントエンドインターフェースをバックエンドサービスに接続

---

### コンセプトスキル

---

## ★ システム説明
このシステムは**社内専用**に設計されており、3つのユーザーロールをサポートします：

### ロール
- **管理者（Admin）** – 完全なアクセス権
- **マネージャー（Manager）** – 在庫、レポート、スタッフ管理
- **スタッフ（Staff）** – 勤務開始/終了、スケジュール確認

---

## 📦 コア機能
### 1. スタッフ管理
- スタッフメンバーの追加/編集
- ロールと時給の設定

### 2. 勤務時間追跡
- 勤務開始/終了の記録
- シフト記録の保存
- 給与計算用データの準備

### 3. 給与レポート
- 勤務時間の計算
- 期間ごとの給与サマリー生成

### 4. 在庫管理
- 在庫数量の追跡
- 再注文閾値の設定
- 販売に基づく在庫の更新

### 5. メニュー/アイテム管理
- 食べ物と飲み物のアイテム
- 原価と売価の追跡
- カテゴリー分類

### 6. 販売ログ
- 注文の記録
- アイテムを販売に関連付け
- 日次売上の追跡

### 7. レポートダッシュボード
- 日別売上
- 売上上位アイテム
- 労働コスト対売上

## 🗄 データベース設計哲学
### 従う規則：
- 各テーブルは**1つのエンティティ**を表現
- 各行は**1つのレコード**を表現
- 名前の代わりに ID を使用
- **データの重複なし**

---

## 🧾 データベーステーブル（初期設計）
### Staff（スタッフ）
```sql
staff_id (PK)
first_name
last_name
role
hourly_rate
```

### Timecards（タイムカード）
```sql
timecard_id (PK)
staff_id (FK)
clock_in
clock_out
work_date
```

### Payroll（給与）
```sql
payroll_id (PK)
staff_id (FK)
pay_period
hours_worked
total_pay
```

その他
